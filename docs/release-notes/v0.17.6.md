# v0.17.6

## ユーザーにとって良くなったこと

- タグの追加・削除がすぐに反映されるようになりました。以前は操作のたびに投稿一覧を全件再読み込みしていましたが、タグが付いた投稿だけを局所更新するようにしました
- ライトボックスで動画を開くと、終了後に自動的に先頭から繰り返し再生されるようになりました
- バックアップ復元に「マージ」モードを追加しました。既存のアーカイブを残したまま、バックアップ内にある未保存の投稿・メディア・タグだけを追加できます
- 一括取り込みの重複自動停止しきい値の上限を 20 から 999 に引き上げました。大量にアーカイブ済みの一覧を取り込むときに途中で止まりにくくなります

## 修正内容

- バックアップをマージ復元したとき、ZIP 内の `post_tags` が既存タグ ID ではなくバックアップ内の別 ID に紐付く問題を修正
- タグ削除時、フィルタ中のタグを削除した投稿がリストから即座に除外されるよう修正（従来は全件再ロード後に反映）

## 詳細

### タグ付けパフォーマンス

`handleAddTagToPost` / `handleRemoveTagFromPost` は操作ごとに `reloadCurrentArchive()` を呼び出して投稿一覧を全件再取得していました。件数が多いほど体感遅延が大きくなるため、background から返ってきた `PostTagRecord` を使って表示中の投稿の `tags` 配列を React state 上で局所更新するように変更しました。タグの件数などのサマリーは `refreshArchiveMetadata` のみで更新します。

タグフィルタ適用中にそのフィルタタグを削除した場合は、DB の状態と表示条件を合わせるため `removePostFromCurrentPage` で該当投稿をリストから除外します。

### マージ復元

復元前に `clearArchiveData()` を呼ぶ「置き換え」と、既存データを保持して差分のみ追加する「マージ」を選択できます。タグは `normalized_name` を照合し、既存タグがあればその ID を使って `post_tags` を追加します（ZIP 内の ID ではなく既存 ID に統一）。OPFS 上のメディアファイルはパスが一致する場合はスキップします。

マージモードでは破壊的操作がないため確認ダイアログを省略しています。
