# v0.16.2

## ユーザーにとって良くなったこと

- X 上で実際に bookmark / like した直後の auto archive が、保存までつながりやすくなりました
- bookmark 成功後に `Save` のまま残ってしまうケースを修正しました
- 投稿の描画が少し遅れている場面でも、保存対象を見つけられずに取りこぼしにくくなりました

## 修正内容

- X 上の実操作 bookmark / like から auto archive が発火しない不具合を修正
- bookmark 成功レスポンスの `data.tweet_bookmark_put = "Done"` を成功扱いに追加
- XHR interceptor で URL から action を先に保持し、`send()` で request body から `tweet_id` を取得するように修正
- 対象 post の `article` が DOM にまだ無い場合に備えて、短時間の retry を追加

## 詳細

v0.16.1 時点では、設定が有効でも X 上の実操作 bookmark / like から auto archive が動かない経路が残っていました。調査の結果、XHR interceptor が `open()` の時点で action と `tweet_id` を同時に解決しようとしており、body がまだ無い段階では action の保持に失敗していました。そのため `send()` 側でも `tweet_id` を拾えず、`x-post-archive:like-bookmark-action` が発火していませんでした。

今回のリリースでは、XHR では URL から action だけを先に保持し、body が渡る `send()` で `tweet_id` を取る形に修正しました。加えて、bookmark の成功レスポンスとして実際に返ってきた `tweet_bookmark_put` を許可し、対象 post が描画途中でまだ DOM に無い場合にも短い retry で取りこぼさないようにしています。

共有 CDP Chrome 環境で unpacked extension を再読込したうえで、実際の bookmark click により `Save -> Saved` へ遷移することまで確認済みです。

## 含まれる変更

- 実 bookmark / like 操作からの auto archive 経路の修正
- bookmark 成功レスポンス判定の更新
- article 未描画タイミングに対する retry の追加
- shared CDP profile 上での実操作確認
