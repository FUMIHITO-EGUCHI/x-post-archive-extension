# v0.19.0

## ユーザーにとって良くなったこと

- ビューアにキーワード検索を追加しました。投稿本文から部分一致で検索でき、フィルター条件と組み合わせても使えます
- キーワード検索の入力内容がフィルターモーダルを開閉しても保持されるようになりました
- ビューアに設定パネルを追加し、表示まわりの挙動を調整できるようにしました（アクセシビリティと文言も改善）
- バックアップからの復元が「追記型」になり、既存アーカイブを上書きしなくなりました。引用元投稿ID（`quoted_post_id`）も保持されます
- 引用投稿のパーマリンク検出が改善され、一部取りこぼしていた引用元へのリンクを拾えるようになりました

## 修正内容

- viewer: キーワード検索フィールドを追加（`UnifiedFilter` にキーワード条件を組み込み、検索結果にハイライトを適用）
- viewer: キーワード検索の state をフィルターモーダルの開閉を跨いで保持するよう修正 (#12)
- viewer: 設定パネルを新設し、アクセシビリティ属性と文言を見直し
- restore: アーカイブ復元を非破壊（merge）化し、`quoted_post_id` を消さないよう修正
- x/extract: 引用投稿コンテナのパーマリンク候補を追加フォールバックで検出
- 未使用の legacy タグ runtime API を削除

## 詳細

### キーワード検索

ビューアのフィルターモーダルにキーワード入力欄を追加しました。入力した文字列は投稿本文に対して部分一致で評価され、既存のタグ・ユーザー・日付フィルターと AND で組み合わせられます。ヒットしたキーワードは結果一覧でハイライト表示されます。

当初の実装ではフィルターモーダルを閉じると入力内容がリセットされてしまう問題があり、Issue #12 で追跡して修正しました。`UnifiedFilter` の状態管理を見直し、モーダル閉時も keyword を保持するようにしています。

### アーカイブ復元の非破壊化

これまで zip バックアップからの復元は既存レコードを上書きしていたため、復元後に一部のフィールド（特に `quoted_post_id`）が欠落するケースがありました。`restore` の実装を merge ベースに変更し、既存レコードの情報を温存しつつ差分のみ反映するようにしています。

### 引用投稿パーマリンク検出

引用コンテナ側に permalink 用の `<a>` が存在しないレイアウトで引用元 URL を取りこぼす挙動が確認されていました。DOM 抽出の候補セレクタを追加し、container 単位でのフォールバック検出を行うようにしました。

### ビューア設定パネル

ビューアに設定パネルを追加し、表示・挙動のカスタマイズ項目をまとめました。合わせてフォーカス制御やラベル文言などのアクセシビリティ・copy を調整しています。
