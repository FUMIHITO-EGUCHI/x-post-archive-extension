# v0.17.2

## ユーザーにとって良くなったこと

- 引用 post を含む投稿を繰り返し保存したとき、既存の引用リンクが意図せず消えるケースを修正しました
- メンテナンス機能（重複画像クリーンアップ）が、OPFS ファイルが存在しない状況でも途中でエラー停止しなくなりました
- バックアップ復元時に引用リンク（`quoted_post_id`）が失われないよう修正しました
- ビューワーの「アーカイブ管理」操作（リセット・復元）が、常にバックグラウンド経由で安全に実行されるようになりました

## 修正内容

コードレビュー（v0.17.1 フォローアップ）で指摘された 22 件の問題をすべて解消しました。主な修正領域：

- **保存ロジック**: 重複保存時の `quoted_post_id` null 上書き防止、auto_tags の保存ペイロード維持、投稿＋メディアの同一トランザクション書き込み
- **メンテナンス**: OPFS ファイル不在での NotFoundError ハンドリング、失敗した keepRecord のステータスリセット、メディア永続化完了待機
- **インポート**: 記事単位の抽出失敗を catch してインポート継続、クォートのみ outer post の null 扱い回避、GraphQL キャッシュへのエントリ上限追加
- **viewer**: stale な `requestPostsPage` レスポンスを無視、video lightbox の OPFS 読み取りキャンセル時の object URL 解放
- **ランタイム**: タイムアウトタイマーの確実なクリア、viewer からのアーカイブ管理をバックグラウンド経由に統一
- **DB**: `listPostIdsByDateFilter` / `listPostIdsWithZeroEngagementCounts` の IndexedDB/Dexie API 最適化、ObjectStore キャッシュの無効化対応

## 詳細

v0.17.1 のコードレビューで発見された内部品質上の問題 22 件をまとめて修正したリリースです。

ユーザーから見て最も影響が大きいのは、引用 post を含む投稿の重複保存時に `quoted_post_id` が消えてしまうケースと、OPFS ファイルが存在しない状況でメンテナンス処理が途中停止するケースの修正です。

それ以外は、インポートの堅牢性向上・メモリリーク防止・型安全性改善など内部品質の修正が中心です。ユーザー向けの新機能追加はありません。
