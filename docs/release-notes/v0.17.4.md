# v0.17.4

## ユーザーにとって良くなったこと

- ランダム表示に引用ポストが混ざらなくなりました
- 設定画面の統計（投稿数・メディア数など）の表示が大幅に速くなりました
- 著者フィルターを適用したときの応答が速くなりました
- 大量の投稿を保存している場合のページ送りが速くなりました（特に「保存日時」「投稿日時」順）
- 一括タグ付けのプレビューキャンセル時に余分なタグが作成されてしまうバグを修正しました
- 引用ポストを再取得した際に、引用元の投稿 ID が正しく保持されるようになりました

## 修正内容

- **ランダム表示**: 引用ポストをランダム表示の候補から除外（`34c8721`）
- **統計表示の高速化**: 設定画面の投稿数・メディア数・タグ数を全件スキャンではなく `count()` で取得（P1）
- **著者フィルターの高速化**: `x_username` インデックスを追加し、著者一覧取得とフィルタリングをインデックスクエリに変更（P2/P3）
- **ページ送りの高速化**: `saved_at` / `posted_at` 降順で keyset pagination を採用し、大きなオフセットでの IndexedDB カーソル前進を排除（P4）
- **ランダムソートのメモリ削減**: 全 ID ロードをやめ、ページ分だけ部分シャッフルで取得（P5）
- **dead code 削除**: 未使用の `posts/list` エンドポイントを削除（P6）
- **タグ保存の原子化**: 新規投稿保存と auto-tag 割り当てを同一トランザクション内で実行（P7）
- **メディア checksum**: OPFS 書き込み時に SHA-256 チェックサムを計算して `MediaRecord` に保存（P8）
- **OPFS クォータエラー**: クォータ超過を他のエラーと区別して warn レベルでログ出力（P9）
- **タグプレビューバグ**: 一括タグ付けプレビュー時に余分なタグレコードが作成されないよう修正（`830a99c`）
- **引用ポスト再取得**: 引用ポストの `x_post_id` が再取得後も保持されるよう修正（`7b84d12`）
- **アーカイブ復元**: 復元パスから `..` を除去しセキュリティを強化。リフェッチメッセージの型を厳格化（`edab71b`）

## 詳細

パフォーマンス改善を中心としたリリースです。

設定画面の統計表示・著者フィルター・ページ送りの 3 箇所で全件スキャンが行われており、投稿数が増えると線形に遅くなっていました。今回、これらをすべて IndexedDB のインデックスクエリまたは `count()` に置き換えました。`x_username` インデックスの追加（DB schema v14）により著者関連の処理が大幅に改善されています。

ページ送りは `saved_at` / `posted_at` の降順に限り keyset pagination を導入しました。従来の `offset(N).limit(M)` は N 件目まで毎回カーソルを進める必要があり大量データで著しく遅くなっていましたが、前のページの末尾値を cursor として使うことでこの問題を解消しています。

メディア保存時に SHA-256 チェックサムを計算して `MediaRecord.checksum` に記録するようになりました。既存レコードは `null` のまま互換性を保ちます（将来のエクスポート/インポート検証に備えた準備）。
