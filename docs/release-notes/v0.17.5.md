# v0.17.5

## ユーザーにとって良くなったこと

- 15GB を超える大きなバックアップでも復元がタイムアウトせず完了するようになりました
- 復元中に進捗バーが正しく更新されるようになりました

## 修正内容

- バックアップ復元を Background Service Worker 経由ではなく Viewer ページで直接実行するように変更
- 10分タイムアウト（`RESTORE_RUNTIME_TIMEOUT_MS = 600000ms`）の制約を排除
- 復元開始前に OPFS staging ファイルへのコピー（最大 15GB 超）が不要になりパフォーマンス改善
- `archive/restore` ランタイムメッセージ型と関連コードを削除

## 詳細

v0.17.4 以前では、バックアップ復元は Background Service Worker への `sendMessage` を介して実行していました。Viewer 側で 10分のタイムアウトが設定されており、15.6GB など大容量バックアップの処理中にタイムアウトが発生してエラーになっていました。さらに、タイムアウト後の catch ブロックが staging ファイルを削除してしまい、Background がまだ読み取り中に `NotFoundError` が発生するという二重の問題がありました。

今回は、復元処理を Viewer ページで `importArchiveBackupZip(file, onProgress)` を直接呼び出す形に変更しました。Viewer は通常タブとして動作するため Service Worker のライフサイクル制限を受けず、タイムアウトが発生しません。また、事前の OPFS staging コピーも不要になりました。

## 含まれる変更

- `settings-archive-maintenance-panel.tsx`: `requestRestoreArchive` を削除し `importArchiveBackupZip` を直接呼び出すよう変更。staging 書き込み・クリーンアップを削除
- `client.ts`: `requestRestoreArchive` 関数と `RESTORE_RUNTIME_TIMEOUT_MS` 定数を削除
- `handle-runtime-message.ts`: `archive/restore` ハンドラーとバリデーターを削除
- `types/runtime.ts`: `RestoreArchiveMessage` / `RestoreArchiveResponse` 型と union 参照を削除
