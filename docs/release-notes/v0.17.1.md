# v0.17.1

## ユーザーにとって良くなったこと

- エンゲージメントゼロ（いいね・RT 0件）の投稿でも、保存後にエンゲージメント数が正しく取得・更新されるようになりました
- 一括インポート時、タイムラインの途中で取得が止まってしまうケースを改善しました
- 引用 post のリンクが遡って正しく補完されるようになりました
- メディア（画像）の重複保存が抑止され、ストレージを無駄に消費しなくなりました

## 修正内容

- エンゲージメントゼロ投稿の refetch 経路を追加（zero-engagement refetch recovery）
- CDP 共有プロファイル環境での zero-engagement refetch の復旧対応
- 一括インポート時のタイムライン走査改善（`fix: improve bulk import timeline traversal`）
- 引用 post のリンク遡及補完（`fix: backfill quoted post linkage`）
- canonical Twitter 画像メディアの重複排除（`fix: dedupe canonical twitter image media`）

## 詳細

v0.17.0 に含まれた bulk import / archive-followups 機能に対する後続修正をまとめたリリースです。

エンゲージメントゼロの投稿（いいね・RT・返信が 0 件）は、APIレスポンスで engagement フィールドが省略されるケースがあり、保存後の refetch で数値が正しく更新されない問題がありました。今回、ゼロ値も対象にした refetch 経路を追加し、CDP 共有プロファイル環境でも動作することを確認しています。

一括インポートでは、タイムライン走査が途中で止まっていたケースを修正し、引用 post の紐付けを遡って補完するロジックも追加しました。あわせてメディアの重複保存も抑止しています。
