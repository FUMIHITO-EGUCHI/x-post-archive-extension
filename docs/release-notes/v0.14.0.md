# v0.14.0

## ハイライト

- X のいいね一括取り込みで、メディア付き投稿の取りこぼしを減らす改善を追加
- likes import の調査用 trace / debug log を追加
- viewer 設定から拡張状態を完全初期化できるよう更新

## 変更内容

このリリースでは、likes import の安定性改善を中心に更新しました。
メディア描画待ちの再評価や richer snapshot の再キューに対応し、画像付き投稿でも保存時点で `mediaCount: 0` になってしまうケースの追跡と抑制を進めています。
あわせて、問題調査をしやすくする debug log と補助スクリプトを追加し、設定画面から OPFS・IndexedDB・ログ・軽量設定をまとめて初期化できるようにしました。

## 含まれる更新

- likes import で media hint を見ながら再評価する待機ロジックを追加
- 同一 `x_post_id` に対して richer snapshot を優先して保存キューへ載せ直すよう更新
- `traceId` 付きの debug log と `logs/clear` runtime message を追加
- inspect 対象投稿 ID を `browser.storage.local` / `localStorage` / URL query から指定可能に
- 調査用の DOM scan / DB summary / log report スクリプトを追加
- settings から拡張機能の状態を完全初期化できるよう更新
