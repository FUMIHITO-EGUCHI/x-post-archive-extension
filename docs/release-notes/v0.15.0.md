# v0.15.0

## ハイライト

- 引用投稿を、主投稿と引用元の 2 件に分けて保存できるよう更新
- viewer で引用元投稿をネスト表示できるよう更新
- 引用投稿に built-in auto tag `引用` / `quoted` が付くよう更新
- X の quoted permalink 取得を main world 側のアノテーション経由に切り替え、テキストのみ引用投稿でも保存できるよう改善

## 変更内容

このリリースでは、引用投稿の保存と表示を扱えるようにする更新を中心に追加しました。
主投稿の中に引用元が入っているケースでも、保存単位は 1 投稿ずつ保ちつつ、主投稿から `quoted_post_id` で引用元を参照できるようにしています。
あわせて viewer 側では引用元を小さなネストカードとして表示し、保存時点の文脈を追いやすくしました。
タグ付けについても、引用元を持つ投稿には built-in auto tag を付けるようにし、あとから引用投稿だけを絞り込みやすくしています。

quoted permalink の取得では、content script の isolated world から React Fiber を直接読まない構成へ変更しています。
main world の content script が quoted container に `data-xpa-quoted-permalink` を付与し、抽出側はその属性を読むことで、テキストのみの引用投稿でも安定して引用元 ID を取得できるようにしました。

## 含まれる更新

- `SavePostInput` / `PostRecord` に `quoted_post_id` を追加
- built-in auto tag に `quoted` / `引用` を追加
- Dexie schema version を v10 に更新
- quoted post 保存時に、引用元を先に保存してから主投稿へ `quoted_post_id` を付与する 2 段階フローへ変更
- likes import でも同じ 2 段階保存フローを使うよう更新
- 主投稿の本文・画像抽出時に quoted container を除外し、引用元メディアの誤保存を抑制
- viewer で `quoted_post` を hydration してネスト表示するよう更新
- quoted permalink 調査と保存確認のための CDP 補助スクリプトを追加

## 補足

- 今回の更新で確認できているのは、少なくともテキストのみ引用投稿 1 件での保存成功と viewer ネスト表示です
- 引用元の `reply_count` / `repost_count` / `like_count` の抽出精度は、今回のリリース範囲には含めていません
