# v0.16.0

## このバージョンでよくなること

- X のブックマーク欄から、いいね欄と同じ感覚で投稿をまとめて保存できるようになりました。
- いいねやブックマークを付けた投稿を、自動でアーカイブへ保存できるようになりました。
- 一覧を投稿者でも絞り込めるようになり、タグと組み合わせて探しやすくなりました。
- 投稿ごとのタグ編集がしやすくなり、設定画面も整理されて使いやすくなりました。
- ライト / ダークテーマの切り替えに対応し、設定やモーダルまわりの見やすさも改善しました。

## 詳細な更新内容

このリリースでは、保存の入口を増やしつつ、あとから探すための絞り込みやタグ操作、設定画面の使い勝手をまとめて改善しました。

保存まわりでは、`x.com/i/bookmarks` に一括保存 UI を追加し、ブックマーク済み投稿を自動スクロールしながらまとめて保存できるようにしています。保存した投稿には `bookmarked` / `ブックマーク` の built-in tag が付くため、あとからブックマーク由来の投稿だけを見返しやすくしました。あわせて、X 上でいいねやブックマークを付けた直後に自動保存する設定も追加しています。

閲覧まわりでは、一覧の投稿者フィルタを追加し、タグ絞り込みとの AND 条件で探せるようにしました。投稿ごとのタグ編集は、一覧上から直接開けるタグピッカーに置き換え、追加や削除を短い操作で行えるようにしています。設定画面も、基本設定 / タグ管理 / 自動タグ変換 / バックアップ / ログのタブに整理しました。

UI 面では、ライト / ダークテーマ切り替えに対応し、ダークモード時の文字コントラストや選択状態の見え方を改善しました。モーダルや設定ナビゲーション、タグ操作まわりのキーボード操作とフォーカス管理も見直し、アクセシビリティを強化しています。

内部的には、古い IndexedDB 状態を持つ環境でも保存処理が止まりにくいように防御を追加し、未使用になっていた旧タグ runtime API も整理しました。さらに、CDP 付き Chrome を安定して起動するための補助スクリプトを追加し、実ブラウザ確認や調査の再現性も上げています。

## 含まれる更新

- `/i/bookmarks` 用の一括保存 UI を追加
- 保存時の built-in tag に `bookmarked` / `ブックマーク` を追加
- いいね / ブックマーク時の自動保存設定を追加
- 一覧の投稿者フィルタを追加
- 投稿ごとのタグピッカー UI を追加
- 設定画面をタブ構成に整理し、タグ管理と自動タグ変換を分離
- viewer のテーマ切り替えを追加
- ダークモード配色、フォーカス表示、モーダル操作を改善
- 旧 IndexedDB 状態に対する保存処理の防御を追加
- 未使用の legacy tag runtime API を削除
- CDP 起動補助スクリプトと調査用 handoff ドキュメントを追加

## 既知事項

- `posts/tags/add` などの旧 runtime API は削除済みで、現行 UI は `post_tag.add` / `post_tag.remove` 系へ統一されています。
- リリースノート時点で実装が入っているのは、個人用アーカイブとしての保存・検索・閲覧改善が中心であり、X 側との完全同期や解除操作への追従は対象外です。
